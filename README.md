This is a libray that helps the users convert transaction data between different formats, namely: csv, binary and txt. It also has two binary targets, described below.

### ypbank_compare

```
cargo run --bin ypbank_compare -- \
  --file1 <path_to_file> \
  --format1 <format> \
  --file2 <path_to_file> \
  --format2 <format>

# where
<format> :=
| csv
| binary
| txt
```

### ypbank_converter

```
cargo run --bin ypbank_converter -- \
  --input <input_file> \
  --input-format <format> \
  --output-format <format> \
  > output_file.txt

# where
<format> :=
| csv
| binary
| txt
```

## Profiling

It is possible to profile two binary targets using:

**Step 1.** Install `samply` into the `cargo` toolchain, `perf` only works with Linux.
```
cargo install samply
```

**Step 2.** Add the below code to `Cargo.toml`
```
[profile.profiling]
inherits = "release"
debug = true
codegen-units = 1
```

**Step 3.** Build the executable you want, for exmaple `ypbank_converter` you would be able to find it in `./target/profiling/ypbank_converter`

```
cargo build --profile profiling --bin ypbank_converter
```

**Step 4.** Run the profiler:
```
samply record ./target/profiling/ypbank_converter --input ./files/records_example.csv --input-format csv --output-format txt > output.txt
```

You can also save the profiling data, so you can later return to it by adding:

```
-o profiling/profile_converter_1.json
```

You can always return to it with:
```
samply load profiling/profile_converter_1.json
```
